Shell脚本编程

1. 第一个 Shell 脚本

执行方式1:

​ bash myFirstShell #使用指定的解释器来执行这个shell脚本

执行方式2:

​ chmod +x myFirstShell #给shell 脚本文件myFirstShell 添加可执行权限，称为可自行文件

​ ./myFirstShell

文件的第一行通常是 #!/bin/bash 或 #!/bin/sh #! 用于指定这个脚本文件使用什么解释器来运行。如果不写这一行，就使用默认的解释器来执行该脚本。bash 或者sh是执行Shell脚本的“解释器”。如果在文件第一行指定了解释器，那么直接执行该脚本时就会自动使用该解释器来运行该脚本。如果使用方式1来执行shell脚本，那么shell脚本的第一行就失效了。

3.7 文件测试操作

-e 文件存在
-f 文件是一个普通文件(不是目录或设备文件)
-s 我呢见大小不为0
-d 文件是一个目录
-b 文件是一个块设备(软盘、光驱等等)
-c 文件是一个字符设备(键盘，调制解调器，声卡等等)
-p 文件是一个管道
-h 文件是一个符号链接
-l 文件存在，而且是一个符号链接
-s 文件是一个socket
-t 文件(描述符)与一个终端设备相关
-r 文件是否可读(指运行这个测试命令的用户的读权限)
-w 文件是否可写(指运行这个测试命令的用户的学权限)
-x 文件是否可执行(指运行这个测试命令的用户的执行权限)
-g 文件或目录的sgid标记被设置。如果一个目录的sgid标志被设置，在这个目录下创建的文件都属于拥有此目录的用户组，而不是创建文件的用户所属的组。
-u 文件的suid标志被设置。如果一个文件的suid位被设置，它将以文件所有者的权限执行该文件而不是执行它的用户的权限。这通常用于允许普通用户以root或其他用户的身份执行特定的程序。
