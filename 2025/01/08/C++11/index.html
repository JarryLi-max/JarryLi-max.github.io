C++11 新特性

1.新类型

1.1 新增加了 long long 和 unsigned long long，以支持64位整型

1.2 新增了char16_t 和 char32_t，以支持16位和32位的字符表示，均为无符号数

1.3 新增原生字符串

原生字符串不对转义字符进行转义。

2. 统一的初始化

C++11 扩大了用大括号的初始化列表的使用范围，使其可用于所有内置类型和用户定义的类型（即类对象）。使用初始化列表的时候，可以添加=，也可以不添加。

2.3 std::initializer_list

如果类有将模板std::initializer_list作为参数的构造函数，则只有该构造函数可以使用列表初始化形式。我们在使用vector等STL时候，发现它的初始化列表可以是任意长度

std::initializer_list除了可以用作构造函数外，还可以用作函数

2.4. 缩窄

初始化列表语法可以防止缩窄，即禁止将数值复制给无法存储它的数值类型

3. 声明

3.1 auto

C++11 将 auto 用于实现自动类型推导

3.2 decltype

decltype：如果仅仅想推到类型而不产生变量如何处理，关键字decltype将变量的类型声明为表达式指定的类型

尤其是在定义模板的时候很有用，因为模板的参数只有传参后才能确定

3.3 nullptr 和 NULL

在 C++ 中推荐使用 nullptr，因为它的本质是指针；而NULL的本质是0

4.4 weak_ptr

std::weak_ptr是一种智能指针，通常不单独使用，只能和shared_ptr类型指针搭配使用，可以视为shared_ptr指针的一种辅助工具。借助weak_ptr类型指针可以获取shared_ptr指针的一些状态信息，比如有多少指向相同的shared_ptr指针、通过expired()判断shared_ptr指针指向的堆内存是否已经被释放等等，还可以解决shared_ptr循环引用的问题。

6. noexcept

noexcept 是一个新的关键字，用来指定一个函数是否不抛出异常

6. C++11 新增对类的操作

6.1 explicit

C++11 中，explicit 关键字可以应用于类的构造函数模板，来避免隐式类型转换

6.2 default

C++11 允许你显式地请求编译器为类生成特殊成员函数，如默认构造函数、复制构造函数和赋值操作符。通过使用 = default，你可以显式地要求编译器自动生成这些成员函数

6.3 override 和 final 关键字**

override：用于标识一个成员函数是重写基类中的虚函数。如果基类中的虚函数没有正确地被重写，编译器会产生错误；

final：用于标识一个类不能被继承，或标识一个虚函数不能被重写。

7. 仿函数

仿函数（Functors）是可以像函数一样使用的对象。通常，仿函数是重载了 operator() 的类或结构体，使得类的实例可以像普通函数一样被调用

8. lambda 表达式

8.1 lambda 格式

  1. [capture-list] : 捕捉列表,该列表总是出现在lambda函数的开始位置,编译器根据[]来判断接下来的代码是否为lambda函数,捕捉列表能够捕捉上下文中的变量给lambda函数使用,如果没有变量需要捕捉,那[]里面的内容可以不写

  2. (parameters)：参数列表, 与普通函数的参数列表一致,如果不需要参数传递,则可以连同()一起省略

  3. mutable：默认情况下,lambda函数总是一个const函数,(如果以传值方式方式捕获变量,是不可以修改的),但是可以使用mutable可以取消其常量性, 使用该修饰符时,参数列表不可省略(即使没有参数,也需要带上参数列表)(取消 const 属性)

  4. return-type：返回值类型, 用追踪返回类型形式声明函数的返回值类型,没有返回值时此部分可省略,返回值类型明确情况下也可省略,由编译器对返回类型进行推导,所以一般不写返回值

  5. {statement}：函数体, 在该函数体内,除了可以使用其参数外,还可以使用所有捕获到的变量

lambda函数的参数列表和返回值类型都是可选部分,但捕捉列表和函数体是不可省略的，所以最简单的lambda为**[]{}**

9. 左值、右值、移动语义、完美转发

9.1 左值

左值代表的是一个具有持久性、可寻址的对象。左值通常表示一个可以取地址的对象，它的生命周期通常较长。你可以将左值赋给一个变量，也可以将其传递给函数。

特点：左值可以出现在赋值语句的左边（因此名字是“左值”），表示的是一个内存位置。

9.2 右值

右值代表的是一个没有持久性的临时对象，或者是值的表达式。右值通常是一个临时对象，它的生命周期通常很短，通常用于表达式的计算结果。右值不能出现在赋值语句的左边，因为它们不对应一个特定的内存位置。

特点：右值没有特定的内存位置（例如，字面常量、临时对象等），其生命周期通常很短，通常是在表达式计算时创建并立即销毁。

9.3 左值引用和右值引用

1. 左值引用（Lvalue Reference）

左值引用是普通的引用类型，它只能绑定到左值

特性：左值引用只能绑定到左值，并且可以修改绑定的对象。

2.右值引用（Rvalue Reference）

右值引用是 C++11 引入的，它用 && 来表示。右值引用可以绑定到右值，并且它是实现 移动语义 和 完美转发 的关键。

特性：右值引用可以绑定到右值，并且它能够允许我们在不复制的情况下“偷取”右值的资源。

9.4 左值与右值的区别

特性 左值 (Lvalue) 右值 (Rvalue)
生命周期 生命周期较长（通常是变量） 生命周期短暂，通常在表达式计算后立即销毁
是否可以取地址 可以取地址 不能直接取地址（除非是通过 std::move 转换为右值引用）
是否可以出现在赋值语句的左边 可以出现在赋值语句的左边（如：x = y） 不能直接出现在赋值语句的左边
例子 变量、数组元素、解引用：x, arr[0], *ptr变量、数组元素、解引用：x, arr[0], *ptr 临时对象、字面常量、计算结果：x + y, 5, std::move(x)

9.5 move

C++11 引入了 std::move 和 **std::forward**，用于将左值转换为右值引用，从而使得右值引用能够接受原本为左值的对象。这是实现移动语义和完美转发的基础。

std::move 它将一个左值转换为右值引用，使得编译器可以选择调用移动构造函数而不是拷贝构造函数，从而允许转移资源，而不是复制它。

9.6 forward

1. 引用折叠

当多个引用类型结合在一起时，可能会出现引用折叠的情况。具体的规则如下：

  • 所有右值引用折叠到右值引用上，仍然是右值引用，例如A&& &&折叠为A&&。
  • 对于非右值引用之间的结合，折叠将导致左值引用，例如A& &折叠为A&；A& &&折叠为A&；A&& &折叠为A&

规则就是：

如果任一引用为左值引用，则结果为左值引用。否则（即两个都是右值引用），结果为右值引用。

2. 万能引用

模板中的&&以及auto&& , 不代表右值引用，而是万能引用，其既能接收左值又能接收右值。

auto&& 是 C++11 引入的一个万能引用（也叫转发引用），它根据绑定的对象的值类别（左值或右值）来推导类型。根据引用折叠规则，当 auto&& 绑定到一个左值时，它会推导成左值引用，而当它绑定到右值时，则推导成右值引用。

3. 完美转发

完美转发的作用是保留函数参数的原始值类别（左值或右值），确保在转发给另一个函数时既不会丢失值类别，也不会发生不必要的拷贝或移动，从而实现高效且灵活的参数传递。

10. 隐式类型转换

operator type() 是一种类型转换运算符（Type Conversion Operator），用于实现隐式类型转换。这种运算符允许你定义如何将一个对象转换为特定的目标类型。当你试图将一个对象赋值给另一种类型的变量时，或者作为函数参数传递时，编译器会自动调用这个类型转换运算符进行转换（如果定义了的话）。
